安装

https://docs.djangoproject.com/zh-hans/4.1/intro/install/ 如果在系统范围安装，安装前需要先装好python3.x。 官方教程还建议使用venv虚拟环境的方式，不过我一直弄不太懂venv，就不尝试了。

1python -m pip install Django

检查是否安装成功：

1python -m django --version

1django-admin startproject mysite

项目会创建一个如下图的目录，关于这些文件都是干啥的： 对于我来说 只有urls、settings两个文件大概会用到 https://docs.djangoproject.com/zh-hans/4.1/intro/tutorial01/

启动项目，我们只要运行刚才生成的manage.py

1python manage.py runserver

注意官网说 已经存在的py文件，如果你修改了，将会在请求的时候按照最新的版本运行，但是如果添加新文件，就要重新启动了。

创建模型

所以我想，这又是和数据库杠上了。我好讨厌数据库，因为数据库这东西真的好麻烦…… 需要创建两个模型：问题 Question 和选项 Choice。

  • Question 模型包括问题描述和发布时间。
  • Choice 模型有两个字段，选项描述和当前得票数，还要连接到Question

总之，我们要给polls造一个数据库，先写好polls/models.py，内容如下：

 1from django.db import models
 2
 3class Question(models.Model):
 4    question_text = models.CharField(max_length=200)
 5    pub_date = models.DateTimeField('date published')
 6
 7class Choice(models.Model):
 8    question = models.ForeignKey(Question, on_delete=models.CASCADE)
 9    choice_text = models.CharField(max_length=200)
10    votes = models.IntegerField(default=0)

这个写法应该比较重点（但是文档里面没强调），因为是使用models.py自动生成建表语句，后续操作相关类等东西，。我们尽量模仿吧。表名是首字母大写，字段中间下划线相连。

左边变量名就是字段名，右边xxxField就是字段类型， 有的有必填参数，比如Char需要max_length，Integer默认为0。这里还写了一个外键和cascade删除。

主键(IDs)会被自动创建。

通过这个polls/models.py，django可以得到创建数据库的信息，但是让他创建之前，我们需要把自己的app加入mysite/settings.py的INSTALLED_APPS列表中：

1INSTALLED_APPS = [
2    'django.contrib.admin',
3    'django.contrib.auth',
4    'django.contrib.contenttypes',
5    'django.contrib.sessions',
6    'django.contrib.messages',
7    'django.contrib.staticfiles',
8    'polls.apps.PollsConfig',
9]

这个步骤是下面的创建表的前置步骤，所以必须要做！

然后执行这个语句生成你的app的数据库迁移文件：

运行 makemigrations 命令，Django 会检测你对模型文件的修改，我们刚才修改了polls/models.py

1 python manage.py makemigrations polls

models 的 __str__函数

鉴于返回的这个玩意我们看不懂，可以去models.py里面给每个class增加一个 __str__()函数，就能显示些有用的东西了。

给polls/models.py新增两个def __str__(self):

 1# Create your models here.
 2from django.db import models
 3
 4class Question(models.Model):
 5	...
 6    def __str__(self):
 7            return self.question_text
 8            
 9class Choice(models.Model):
10	...
11    votes = models.IntegerField(default=0)
12    def __str__(self):
13        return self.choice_text

再添加一个was_published_recently(self)函数，可以给Question增加一个“最近发布”的功能：

1from django.utils import timezone
2
3class Question(models.Model):
4	...
5    def was_published_recently(self):
6        return self.pub_date >= timezone.now() - datetime.timedelta(days=1)

去py shell里面看一下，刚刚写的函数有没有效果

1>>> from polls.models import Choice, Question
2# 看看__str__() 有没有效果
3>>> Question.objects.all()
4#<QuerySet [<Question: What's up?>]>
5
6# 试试刚才写的“最近发布？”功能
7>>> q = Question.objects.get(pk=1)
8>>> q.was_published_recently()
9# True
